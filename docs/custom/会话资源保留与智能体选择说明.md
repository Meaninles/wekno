# 会话资源保留与智能体选择说明

## 背景

聊天输入区曾在进入已有会话或首条消息创建会话后，按会话 `last_request_state` 覆盖完整输入状态，导致用户当前选择的智能体、模型被旧会话状态带偏，典型表现为选择“表格分析”后打开/创建会话又显示为“快速问答”。

本次二开调整将“用户当前选择”和“会话引用资源”拆开处理：

- 智能体、模型：作为用户当前选择保留，不再因打开已有会话自动回放旧会话的智能体和模型。
- 会话资源：仅在同一个会话内恢复，包括知识库、知识库文档、上传附件、上传图片、Skill、MCP、网络搜索状态。
- 新建会话或离开会话：资源恢复默认空态，避免旧会话引用泄漏到新对话；智能体和模型继续保留用户当前选择。

## 前端实现

核心状态方法位于 `frontend/src/stores/settings.ts`：

- `resetConversationScopedState()` 默认保留智能体和模型，清空会话资源。
- `applyConversationResourceState()` 只恢复资源字段，不覆盖 `agent_id`、`agent_enabled`、`model_id`。
- `captureConversationScopedState()` 仍负责创建会话和发送消息时捕获请求状态，供同一会话后续恢复。

桌面端实现位于：

- `frontend/src/components/Input-field.vue`
  - 图片上传增加 `setUploadedImages`、`getUploadedImages`、`clearUploadedImages` 暴露方法。
  - 发送消息后不再清空图片和附件，保证同一会话可继续复用。
- `frontend/src/views/creatChat/creatChat.vue`
  - 创建新 session 时写入 `last_request_state`。
  - 首条消息的附件和图片写入会话 draft。
- `frontend/src/views/chat/index.vue`
  - 进入会话时先清空资源，再恢复该 session 的资源态和内存 draft。
  - 首条消息从 `/platform/creatChat` 跳转到 `/platform/chat/:id` 后，立即重新应用 draft 资源态，避免 Skill、图片等在切路由时丢失。

移动端实现位于 `frontend/src/custom/modules/mobile/views/MobileChat.vue`：

- 打开会话时只调用 `applyConversationResourceState()` 恢复资源字段。
- 同一会话内保留 `pendingImages`、`pendingAttachments`。
- 新建会话时清空资源，但保留当前智能体和模型。

## 行为约定

同一会话内应保留：

- 上传附件和图片。
- 已选择知识库、知识库文档、标签。
- 已选择 Skill、专业 Skill、MCP 服务。
- 网络搜索开关状态。

跨会话或新建会话应恢复默认资源空态：

- 不带入上一个会话的附件、图片、Skill、知识库、文档、联网状态。
- 不自动把当前智能体或模型改成旧会话记录。

## 已验证场景

桌面端：

- 上传 `resource-test.png` 和 `resource-test.csv` 发送后，同一会话输入区保留图片 1 张、附件 1 个。
- 点击“新对话”后资源清空，返回原会话后资源恢复。
- 选择“表格分析”发送后，输入区仍显示“表格分析”，不会跳回“快速问答”。
- 选择 Skill“引用生成器”发送后，输入区保留 Skill chip；新对话清空，返回原会话恢复。

移动端：

- 当前选择为“表格分析”时打开和新建会话，不会自动切回“快速问答”。
- 上传图片和 CSV 后发送，同一会话保留“图片 1 / 附件 1”。
- 新建会话后图片和附件清空，返回原会话后恢复。

## 注意事项

上传图片和附件的 `File` 对象只能在当前浏览器运行上下文内通过内存 draft 恢复。浏览器整页刷新后，出于浏览器安全限制，前端无法从本地文件路径自动重建原始 `File` 对象；知识库、知识库文档、Skill、MCP、网络搜索等服务端 `last_request_state` 字段仍可恢复。
