# WeKnora 用户使用指南

本文面向平台使用者、知识库维护者、共享空间管理员和系统管理员，覆盖当前系统中可以通过界面完成的主要功能。若你只想快速完成某个需求，先看“需求索引”；若你要排查权限、共享、模型、解析或集成问题，优先看对应章节的“注意事项”和“排错”。

## 核心概念

WeKnora 是一个面向知识管理和智能体应用的平台，核心对象包括：

| 概念 | 说明 |
| --- | --- |
| 租户 / 工作区 | 资源归属单位。知识库、智能体、模型、数据源、MCP 服务等都属于某个租户。用户可以加入多个租户并在左下角账号菜单切换。 |
| 用户 | 登录系统的账号。用户在每个租户内有一个租户角色，在共享空间内也可以有另一个空间角色。 |
| 知识库 | 存放文档、FAQ、Wiki 页面、向量索引、关键词索引、图谱等信息源。对话和智能体可以检索知识库。 |
| 智能体 | 对话运行配置，决定使用什么模型、提示词、知识库、数据库、联网搜索、MCP、技能、多模态和工具。 |
| 会话上下文 | 用户在某个会话中临时选择的知识库、文档、标签、MCP、技能、联网开关、图片和附件。它只在当前会话内恢复，不会自动带到新会话。 |
| 技能 | 给智能体补充能力或提示词。轻量技能是预置提示词；专业技能是可导入的技能包。 |
| 共享空间 | 跨租户协作空间。共享空间本身不拥有知识库或智能体，只记录“某资源共享给某空间”的关系。 |
| 数据源 | 有两类：一类是同步到知识库的外部内容源；另一类是供数据分析、通用智能体和文档处理智能体查询的 MySQL/PostgreSQL 数据库源。 |
| MCP 服务 | Model Context Protocol 工具服务，用来把外部工具接入智能体。 |
| Principal | API、嵌入、IM 等外部入口中的终端调用者身份，用于隔离会话和 MCP OAuth 授权，不等同于租户 RBAC 权限。 |

## 权限与角色

系统有两套权限：租户内角色和共享空间角色。它们是正交的。

### 租户角色

| 角色 | 适合对象 | 主要能力 |
| --- | --- | --- |
| viewer | 只查询、只提问的用户 | 可读取可见资源、发起对话；不能创建或修改资源。 |
| contributor | 内容维护者、智能体配置者 | 可创建和管理自己创建的知识库、智能体、技能等资源；对别人资源通常只读。 |
| admin | 工作区管理员 | 可管理租户内大部分资源、成员、模型、解析器、存储、向量库、MCP、数据源等基础设施。 |
| owner | 工作区所有者 | 拥有 admin 权限，并可管理 API Key、Owner-only 配置和删除租户等高风险操作。每个租户通常只有一位 owner。 |

API Key 请求在所属租户内固定视为 admin，但不会获得 SystemAdmin 权限，也不能替代人类账号执行平台级超管操作。

### 共享空间角色

| 角色 | 主要能力 |
| --- | --- |
| 管理员 | 管理空间设置、成员、邀请码、加入申请，并可取消空间内任意共享关系。 |
| 编辑者 | 可把自己的资源共享到空间；对以可写权限共享的知识库可执行编辑类操作。 |
| 只读 | 只能查看和使用空间中共享给自己的资源。 |

共享空间不改变资源归属。一个知识库共享给某空间后，知识库仍属于源租户；成员能否写入，取决于共享权限、空间角色以及源资源的保护规则。

## 账号、租户与工作区

### 登录和注册

系统支持普通账号登录、注册、邀请注册链接、OIDC 登录，以及已配置时显示的统一身份认证登录入口。公开注册是否可用由平台配置决定；如果系统设置为仅邀请注册，登录页不会允许直接自助注册。

登录后，前端会保存访问令牌和刷新令牌。访问令牌过期时会自动刷新；刷新失败会清理本地登录状态并跳回登录页。

### 切换工作区

用户加入多个租户后，可以通过左下角账号菜单切换工作区。切换后，前端会在请求中带上当前工作区的 `X-Tenant-ID`，使知识库、智能体、会话、模型和设置都运行在目标租户下。

如果切换后看不到资源，通常是以下原因：

- 当前用户不是目标租户的有效成员。
- 资源属于另一个租户，未通过共享空间共享给当前用户。
- 当前角色低于该操作要求，例如 viewer 想创建资源。
- 浏览器本地状态过旧，刷新页面或重新登录后再试。

### 邀请和成员管理

owner/admin 可以在设置中心管理租户成员。邀请可以被接受或拒绝；成员角色变更会写入审计日志。若用户离开租户，无法再看到该租户内资源，除非资源又通过共享空间向其当前租户共享。

## 使用指南共享知识库

本系统会维护一个名为“使用指南”的共享知识库，用来放置系统说明文档。它的目标是让所有现有用户和新增用户都能自动看到，不依赖首次登录教学流程。

规则如下：

- 系统启动时会确保“使用指南”共享空间和知识库存在，并把活跃用户加入到该共享空间。
- 新用户注册、统一身份同步或用户信息 provisioning 时，也会自动加入该共享空间。
- 用户不需要在首次登录教学中手工加入；即使教学弹窗重复出现，也不会影响用户是否可见“使用指南”。
- 大多数用户对该知识库应以只读方式使用，用来检索文档、查看操作说明。
- 管理员负责维护其中的 Markdown 文档，例如本指南和智能体开发指南。

如果你是管理员，想更新“使用指南”内容，建议直接在该知识库中上传或替换 Markdown 文档。为了避免普通用户误改，维护权限应只授予实际负责文档维护的管理员或编辑者。

## 对话

对话是用户使用知识库和智能体的主要入口。左侧菜单点击“新对话”进入对话界面。

### 基本流程

1. 选择智能体。可以选择内置智能体、自建智能体或通过共享空间获得的智能体。
2. 选择知识库、文档、标签、MCP 服务、技能等上下文。不同智能体允许的选择范围不同。
3. 在输入框提问。支持普通文本，也支持在启用后上传图片、音频或附件。
4. 等待流式回答。回答过程中可以停止生成；如果中断或刷新，系统会尝试恢复未完成的流式响应。
5. 查看引用、工具结果、来源时间线和消息反馈。

### 对话上下文保留规则

电脑 Web 端和移动端使用同一套规则：

- 打开、切换或返回同一个会话时，会恢复该会话上次提问时使用的知识库、知识库文档、标签、MCP 服务、轻量技能、专业技能和联网搜索开关。
- 同一会话内未发送的本地图片和附件会在当前前端运行期间保留；如果刷新浏览器或重新打开页面，浏览器不会把本地 `File` 对象持久化，需要重新选择。已经发送过的图片和附件会保留在消息记录中。
- 打开历史会话不会把当前已选智能体和模型自动改回历史值；系统会保留你当前选择的智能体和模型，避免旧会话把输入区切回“快速问答”。
- 新建会话会清空知识库、文档、标签、MCP、技能、联网搜索、待发送图片和附件等会话资源，但保留当前选择的智能体和模型。
- 从知识库、智能体或带 `agent_id`、`kb_ids`、`file_ids` 的链接进入新会话时，链接中的初始上下文会覆盖新会话的空上下文。

### 选择知识库和文档

对话中可选择多个知识库，也可以选择某些文档或标签。若智能体配置了“仅在显式 @ 提及时检索知识库”，则系统不会自动检索知识库，只有当你在问题里明确 @ 某知识库、文档或资源时才会检索。

常见用法：

- “@使用指南 如何创建智能体？”让回答优先使用“使用指南”知识库。
- 选择某个知识库后提问“总结这些制度的审批流程”，系统会从该知识库召回相关片段。
- 选择某个标签后提问，适合只检索某类文档。
- 对共享知识库提问时，系统会根据共享关系解析源租户资源，但你仍只能访问自己有权限看到的内容。

### 回答引用和来源

当智能体使用知识库、Wiki、网络搜索或工具时，回答中会显示引用来源。你可以展开引用查看：

- 引用了哪些文档、网页或 Wiki 页面。
- 命中的文本片段。
- 来源知识库、来源文档、标签、渠道。
- 工具调用或检索过程的摘要。

如果回答没有引用，可能是智能体没有启用知识库检索、问题没有命中、知识库未完成解析，或当前问题被判定为闲聊/非检索意图。

### 图片、音频和附件

是否能上传图片、音频或文件，由当前智能体配置决定：

- 图片上传需要智能体启用图片上传，并选择支持视觉能力的 VLM 模型和图片存储。
- 音频上传需要智能体启用音频上传，并选择 ASR 模型。
- 文档附件会作为当前问题的上下文发送给智能体；文档处理、表格分析、数据分析、通用智能体更适合处理附件。
- 表格分析智能体默认只允许 CSV、XLSX、XLS 等表格文件；移动端和电脑端都会按当前智能体的文件类型限制过滤附件。
- 嵌入渠道和 IM 渠道是否允许文件上传，还要看渠道配置。

### 会话管理

会话列表支持查看、置顶、取消置顶、删除、未读提示、生成标题和按来源过滤。会话来源可能是网页、API、浏览器扩展、IM 或嵌入渠道。Web 端侧边栏和移动端会话抽屉都会在有新助手回复时显示未读提示；打开对应会话后会自动标记为已读。

通过 API、嵌入、IM 创建的会话会按对应 Principal 隔离。也就是说，不同外部用户、不同嵌入访客、不同 IM 用户通常不会看到彼此的会话。

### 对话分享

电脑 Web 端对话页右上角和移动端聊天标题区域都提供分享入口。点击后会为当前会话生成一个只读链接：

```text
/share/chat/:token
```

分享链接不区分电脑端和移动端，打开后会按视口自动使用桌面或移动布局。查看分享页必须先登录 WeKnora，但不要求查看者拥有原会话、原租户或原知识库权限。

分享内容是创建链接当时的消息快照。它会展示消息文本、图片、附件、结构化分析图表和通用智能体产物下载入口；不会展示输入框、继续提问、引用跳转、来源时间线跳转、agent steps 或原会话跳转。原会话后续新增或删除消息，不会自动同步到已经创建的分享链接。

### 消息反馈

助手回答下方可进行点赞、点踩或取消反馈。反馈会记录在当前租户下，用于后续质量分析或人工改进知识库、提示词和智能体配置。

### 移动端 Web

移动端入口通常是同域名下的 `/mobile/`，开发环境可通过 `http://localhost:18081/mobile/` 访问。移动端支持聊天、会话抽屉、智能体和模型选择、知识库/文档/标签上下文、联网搜索、技能选择、图片和附件上传、对话分享、账号设置，以及基础知识库文档管理。

移动端“知识库管理”可以查看知识库和文档列表，上传当前解析器支持的文件，下载文档，删除文档，并轮询解析状态。移动端不是完整后台，知识库创建、高级解析配置、共享空间、智能体编辑和系统设置仍建议在电脑 Web 端完成。

## 知识库

知识库位于左侧“信息源”。列表中会展示我的知识库、同租户他人知识库、通过共享空间获得的知识库、收藏和最近使用资源。你能看到哪些按钮，取决于角色和资源归属。

### 知识库类型和能力

当前主要类型包括：

| 类型 | 适用场景 | 关键能力 |
| --- | --- | --- |
| 文档知识库 | 文档、网页、Markdown、图片、音频等资料检索 | 向量检索、关键词检索、Wiki、图谱、问答推荐、多模态解析。 |
| FAQ 知识库 | 标准问答、客服问答、制度问答 | 标准问题、相似问题、反例问题、多个答案、分类、批量导入导出、推荐问题。 |
| Wiki 能力 | 长文档结构化知识、页面式导航和概念关系 | 页面、文件夹、索引、页面链接图谱、来源文档、问题标记和自动修复。 |

同一个文档知识库可以启用多种索引策略。向量/关键词用于 RAG 检索，Wiki 用于页面化知识组织，图谱用于实体关系抽取。Wiki 页面链接图谱和设置中的实体关系图谱不是同一个概念。

### 创建和配置知识库

点击“信息源”中的新建按钮创建知识库。创建时通常需要设置：

- 基本信息：名称、描述、类型。
- 索引策略：向量检索、关键词检索、Wiki、图谱等。文档知识库至少选择一种索引能力。
- 模型：摘要/问答模型、Embedding 模型、Wiki 合成模型、VLM 模型、ASR 模型等。启用向量或关键词检索时需要 Embedding 模型；纯 Wiki 时 Embedding 可选。
- 向量库：选择可用向量存储。
- 解析器：选择文档解析引擎，可针对 PDF 等文件调整解析方式。
- 存储：选择本地、MinIO、COS、S3、OSS、TOS、KS3、OBS 等已配置存储。
- 分块：设置 chunk size、overlap、父子分块、分块策略、token 限制和语言。
- 多模态：启用图片解析、选择 VLM 模型和图片处理方式。
- ASR：启用音频转写、选择 ASR 模型。
- 图谱：配置实体关系抽取标签和图谱能力。
- 外部数据源：配置飞书、Notion、语雀或 RSS 同步。
- 共享：把知识库共享到共享空间并设置只读或可写。

### 上传和导入资料

知识库详情页支持以下入口：

| 入口 | 说明 |
| --- | --- |
| 上传文档 | 选择本地文件上传到知识库。 |
| 上传文件夹 | 选择本地文件夹批量上传，系统会过滤隐藏文件、视频等不支持类型。 |
| URL 导入 | 输入网页或文件 URL，系统抓取并解析。 |
| 在线编辑 / 手动创建 | 以 Markdown 草稿形式创建内容，发布后进入解析。 |
| 继续添加 | 在上传确认窗口中继续追加文件或 URL。 |

上传确认窗口会显示将要入库的文件列表，并允许覆盖本次解析参数，例如解析器、分块、多模态、ASR、问题生成和图谱配置。对于包含图片的资料，需要启用多模态并选择 VLM 模型；对于音频资料，需要启用 ASR 并选择 ASR 模型。

### 文档状态

文档解析状态包括 pending、processing、finalizing、completed、failed、deleting、cancelled 等。卡片或列表中会显示当前状态，解析中和失败时可以打开解析时间线查看阶段耗时和错误。

常见操作：

- 预览：查看原文、解析文本、图片或音频播放器。
- 下载：下载原始文件。
- 重解析：使用当前或覆盖后的知识库配置重新解析。
- 取消解析：终止正在处理的任务。
- 删除：删除文档和相关索引。
- 批量删除/批量重解析：在批量模式选择多个文档后执行。
- 移动到其他知识库：可选择复用向量或重新解析。复用向量更快，但目标知识库解析配置差异较大时建议重解析。

### 标签和筛选

知识库内可以创建标签，对文档进行分类。标签可用于列表筛选、对话检索范围限定和知识维护。FAQ 知识库也有分类标签，但 FAQ 标签作用在问答条目上。

### 知识库共享

知识库可共享到一个或多个共享空间，每个共享关系独立设置权限：

- 只读：空间成员最多只能查看和检索。
- 可写：空间内管理员或编辑者可以在共享规则允许的范围内编辑。

共享不改变知识库归属。取消共享后，空间成员将不再通过该共享空间看到该知识库。

## FAQ 知识库

FAQ 知识库适合标准问答、客服问答、制度解释、产品问答和固定答案场景。

### FAQ 条目结构

每条 FAQ 包含：

- 标准问题：主问题，必填。
- 相似问题：最多 10 个，用于提高匹配召回。
- 反例问题：最多 10 个，用于减少误命中。
- 答案：最多 5 个，必填。智能体会根据策略引用或直接回答。
- 标签/分类：用于管理和筛选。
- 启用状态：关闭后不参与匹配。
- 推荐状态：可作为推荐问题来源。

### 新增、编辑和搜索

进入 FAQ 知识库详情页后，可以新增条目、编辑条目、切换启用状态、切换推荐状态、按标签筛选、按问题搜索。只读用户只能查看；有编辑权限的用户可以修改。

### 导入和导出

FAQ 支持导入 JSON、CSV、Excel。导入模式包括追加和替换。导入前会预览前几条数据，导入过程中显示进度，完成后显示总数、成功、失败和跳过数量；失败记录可下载原因文件。

CSV/Excel 常用列名：

- `问题`、`standard_question` 或 `question`
- `机器人回答` 或 `answers`
- `相似问题` 或 `similar_questions`
- `反例问题` 或 `negative_questions`
- `标签`、`分类` 或 `tag_name`
- `tag_id`

多个相似问题、反例问题或答案可用 `##` 分隔。系统也支持下载示例 CSV。

### 批量操作

选择多个 FAQ 条目后，可以批量改标签、批量启用、批量停用、批量推荐、批量取消推荐和批量删除。

### FAQ 回答策略

智能体可配置 FAQ 优先、直接回答阈值和 FAQ 分数加权。当问题高置信命中 FAQ 时，系统可以优先使用 FAQ 答案；当命中不稳定时，建议提高反例问题质量或调高阈值。

## Wiki 知识库

Wiki 适合把大量文档组织成页面、目录和概念关系。启用 Wiki 后，知识库详情页会出现 Wiki 标签页。

### Wiki 能做什么

- 按页面浏览知识。
- 创建和管理文件夹。
- 查看页面来源文档。
- 搜索 Wiki 页面。
- 查看页面链接关系图。
- 查看统计信息。
- 标记页面或内容问题。
- 使用自动修复能力处理部分 Wiki 问题。

### Wiki 和 RAG 的区别

RAG 检索面向“从片段中找答案”，Wiki 面向“把知识整理成页面并沿概念关系阅读”。如果你的需求是精确问答，启用向量/关键词检索；如果你的需求是构建体系化知识、目录和页面关系，启用 Wiki；如果两者都需要，使用混合能力。

### Wiki 图谱和实体关系图谱

Wiki 标签页中的关系图是页面之间的引用/链接关系图。知识库设置中的图谱是基于 LLM 从文本中抽取实体和关系形成的实体关系图。两者用途不同，可以同时存在。

## 知识库外部数据源

知识库设置中的“数据源”用于把外部内容同步到当前知识库。当前界面支持飞书、Notion、语雀和 RSS。

### 配置流程

1. 在知识库设置中进入“数据源”。
2. 选择连接器类型。
3. 填写认证信息，例如 Notion Integration Token、语雀 Token、RSS URL 等。
4. 测试连接。
5. 加载资源树并选择要同步的空间、页面、文件夹或订阅源。
6. 设置同步周期、同步模式和冲突策略。
7. 保存后可手动“立即同步”，也可等待定时同步。

### 同步配置

| 配置 | 说明 |
| --- | --- |
| 同步周期 | 支持 30 分钟、1 小时、6 小时、12 小时、24 小时等。 |
| 同步模式 | 增量同步或全量同步。 |
| 冲突策略 | 外部内容变化时覆盖或跳过。 |
| 删除同步 | 外部资源删除后是否同步删除本地内容。 |
| 凭证管理 | 可替换或移除已保存凭证。 |
| 日志 | 查看每次同步的状态、开始结束时间、成功/失败/跳过数量。 |

只有 admin 及以上角色可管理数据源；viewer 可以查看已同步内容。

## 数据库数据源

左侧“数据源”是给 Claude SDK 类型智能体使用的数据库源，和知识库外部数据源不是同一类功能。当前支持 MySQL 和 PostgreSQL。数据分析智能体必须绑定至少一个数据库源；通用智能体和文档处理智能体可按任务需要绑定。

### 创建数据库源

进入“数据源”，点击新增，填写：

- 名称、描述。
- 类型：MySQL 或 PostgreSQL。
- Host、Port、Database、Username、Password。
- SSL Mode。
- 最大返回行数、最大扫描行数、查询超时。

保存前建议先测试连接。密码会加密存储；如果环境密钥变化导致无法解密，需要重新录入凭证。

### 元数据和安全范围

创建后可以刷新元数据，系统会读取表、视图、字段信息。管理员可以：

- 设置哪些表启用给智能体使用。
- 为字段补充中文描述。
- 标记字段语义类型：维度、指标、时间等。
- 设置敏感级别：无、脱敏、隐藏。
- 查看数据源状态 active/error。

智能体查询数据库时，应先读取 catalog/schema，再生成只读 SELECT 查询。建议限制表范围、行数和扫描量，避免高成本查询。

### 共享数据库源

数据库源可共享到共享空间，权限可设为 viewer/editor。共享后，具备权限的成员可在数据分析、通用智能体或文档处理智能体中绑定该数据源。

## 智能体

智能体位于左侧“智能体”。列表会显示内置智能体、我的智能体、同租户他人智能体、共享可编辑智能体和共享只读智能体。contributor 及以上可创建智能体；能否编辑取决于角色、归属和共享权限。

### 内置智能体

系统提供多种内置智能体，可直接使用，也可以复制后按自己的需求修改。

| 智能体 | 适合场景 |
| --- | --- |
| 快速问答 | 经典知识库问答，适合基于文档/FAQ 快速回答并给出引用。 |
| 简单对话 | 不默认绑定知识库，适合闲聊、写作、联网搜索、临时文件/图片/音频问题。 |
| 智能推理 | ReAct Agent 模式，适合需要多步推理、工具调用、知识库检索、MCP 的任务。 |
| 维基问答 | 面向 Wiki 知识库的问题，适合页面化知识导航。 |
| 数据分析 | 适合连接 MySQL/PostgreSQL 数据源，生成 SQL、分析结构化数据和图表。 |
| 表格分析 | 适合分析知识库或上传的 CSV/Excel 文件，使用 DuckDB SQL 并按需生成结构化图表。 |
| 通用智能体 | 适合同时使用知识库、网络、MCP、技能、数据库和产物生成的复杂任务。 |
| 文档处理 | 适合生成、修改 Word、Excel、PDF、PPT 等办公文档。 |

配置文件中还包含 `builtin-wiki-fixer`（维基修订）内部智能体，用于 Wiki 页面修订流程，普通智能体列表不默认展示。

内置智能体可在有权限时恢复默认配置。恢复会由后端按策略保留或跳过部分模型、数据源、MCP 范围，避免误删环境相关配置。

### 创建智能体

创建时先选择模式和类型：

- 快速问答模式：偏稳定 RAG，配置较少，适合标准知识库问答。
- 智能推理模式：支持工具、MCP、技能、多步推理，适合复杂任务。

智能推理模式下可选择类型预设：

- RAG 问答
- Wiki 问答
- Wiki + RAG 混合
- 文档处理
- 数据分析
- 表格分析
- 通用智能体
- 自定义

类型预设会自动填充提示词、工具、知识库兼容规则和部分默认参数。

### 智能体配置项

智能体编辑器包含以下区域：

| 区域 | 说明 |
| --- | --- |
| 基本信息 | 名称、描述、头像、运行模式、类型。 |
| 提示词 | 系统提示词、上下文模板、意图提示词、问题改写提示词、兜底提示词。 |
| 文档模板 | 文档处理智能体可配置 Word/Excel/PDF/PPT 要求文件和模板文件。 |
| 模型 | 主模型、温度、最大输出、思考模式、最大迭代次数、LLM 调用超时、产物生成。 |
| 对话 | 多轮对话、历史轮数等。 |
| 知识库 | 选择全部、指定或不使用知识库；设置是否仅显式 @ 时检索。 |
| 数据库 | 选择 MySQL/PostgreSQL 数据源。数据分析智能体必填，通用智能体和文档处理智能体可选。 |
| 检索 | Query expansion、问题改写、TopK、关键词阈值、向量阈值、重排 TopK/阈值、FAQ 策略。 |
| 联网搜索 | 启用网络搜索、选择搜索提供商、最大结果数、Claude SDK 原生搜索、网页抓取。 |
| 多模态 | 图片上传、VLM 模型、图片存储、音频上传、ASR 模型。 |
| 工具 | 按工具组选择允许的基础工具、RAG、Wiki、数据分析、数据库等工具。 |
| MCP | 选择全部、指定或不使用 MCP 服务，设置 OAuth 授权等待超时。 |
| 技能 | 选择轻量技能和专业技能。 |
| 共享 | 将智能体共享给共享空间。 |
| 集成 | 查看或跳转配置 IM 渠道和网页嵌入渠道。 |

### 智能体分享和停用

智能体可以共享到共享空间，当前共享权限以只读使用为主。空间成员可在对话中选择共享智能体，但不能直接编辑源智能体。

当前租户可以把共享来的智能体标记为停用。停用只影响当前租户的选择列表，不会改变共享关系，也不影响其他成员。

## 技能

技能位于左侧“技能”，分为专业技能和轻量技能。

### 轻量技能

轻量技能是预置提示词，不要求 `SKILL.md` 格式。适合固定写作风格、审校规范、行业术语约束、输出模板等场景。

创建轻量技能需要填写：

- 技能名称。
- 技能描述。
- 预置提示词。
- 是否启用。

轻量技能可共享到共享空间或指定用户。对话中可以从技能选择器搜索、选择、全选、清空或置顶。

### 专业技能

专业技能是以技能包形式导入的能力，通常包含 `SKILL.md`、引用文件、脚本或模板。系统会显示技能名称、描述、文件数量、是否本地导入、是否系统保留、是否可下载。

专业技能适合更复杂的智能体工作流，例如工具说明、结构化工作方法、行业模板、脚本辅助等。可导入、编辑、下载、共享到共享空间或用户。

### 在对话和智能体中使用技能

有两种使用方式：

- 在智能体配置中固定选择技能，让所有使用该智能体的对话默认带上这些能力。
- 在对话输入区临时选择技能，作为当前会话上下文保存；切换回同一会话会恢复，开启新会话会清空。

如果某个专业技能在选择器里不可见，可能是当前智能体不是通用智能体或文档处理智能体、没有启用专业技能选择、技能未共享给你、技能被禁用或你没有进入正确租户。

## 定时任务

定时任务位于左侧“定时任务”，用于让系统按计划自动向某个智能体提问，并把回答写入真实会话。

### 创建定时任务

创建任务时需要设置：

- 任务名称和描述。
- 是否启用。
- 运行用户。
- 绑定智能体。
- 时区，默认 `Asia/Shanghai`。
- 调度类型：小时、每天、每周、每月。
- 分钟、小时、星期、日期等具体时间。
- 提示词模板。
- 是否启用联网搜索。
- 请求上下文：知识库、文档、标签、MCP 服务、轻量技能、专业技能、图片、附件等。

系统支持变量和提示词模板，并提供渲染预览。建议保存前先预览，确认变量已经被替换成预期内容。

### 运行和查看结果

任务可手动“立即运行”，也可等待计划触发。每次运行会创建运行记录，包括触发方式、状态、渲染后的提示词、会话 ID、消息 ID 和错误信息。成功后可以跳转到生成的会话查看完整回答。

系统的并发策略是已有运行未完成时跳过新的计划触发，避免同一任务重复并发执行。单次运行有较长超时时间，适合报告生成类任务。

### 适合的场景

- 每天生成业务日报、周报、月报。
- 定期检查某个知识库是否有新增问题。
- 定期查询数据库并输出指标摘要。
- 定时联网搜索并汇总行业动态。
- 定时读取指定附件、图片或文档并生成分析。

## 共享空间

共享空间位于左侧“共享空间”。空间用于跨租户共享知识库、智能体、技能和部分数据源。

### 创建、加入和邀请

任意登录用户可创建共享空间，创建者自动成为空间管理员。空间管理员可以生成邀请码、设置有效期、管理成员、审批加入申请和调整空间设置。

用户可以通过邀请码加入空间。如果空间开启审批，加入申请需要管理员处理。

### 共享资源

可共享资源包括：

- 知识库：只读或可写。
- 智能体：供空间成员使用。
- 轻量技能和专业技能：供空间成员选择或配置到智能体。
- 数据库数据源：供数据分析、通用智能体或文档处理智能体使用。

共享空间不拥有资源；资源仍归源租户。取消共享后，成员无法再通过空间看到资源。

### 跨租户写入判断

对他人共享给你的知识库执行写操作时，需要同时满足：

1. 知识库已经共享到你所在的空间。
2. 共享关系允许可写。
3. 你在共享空间内至少是编辑者。
4. 源资源自身允许通过共享路径写入。

因此，“共享为可写”不代表任何空间成员都能无条件修改源租户资源。

## 集成

左侧“集成”包含 IM、网页嵌入、API、Chrome 插件和 ClawHub。API 标签需要 owner 权限；IM 和网页嵌入的管理动作通常需要 admin 权限。

### API 集成

API 页面显示当前工作区的 API Base URL 和 API Key，并提供 Principal 模式配置和 Playground。

调用 API 时常用头：

```http
X-API-Key: sk-xxxxx
Content-Type: application/json
```

也可以使用登录后的：

```http
Authorization: Bearer <jwt>
```

API Key Principal 模式：

| 模式 | 说明 |
| --- | --- |
| tenant | 全租户共用一个终端身份，适合没有外部用户隔离需求的后台集成。 |
| direct_header | 通过 `X-External-User-ID` 传外部用户 ID，只适合可信服务端到服务端调用。 |
| signed_token | 推荐模式。业务后端用 HMAC secret 签发短期 HS256 JWT，通过 `X-External-User-Token` 传给 WeKnora。 |

Principal 主要隔离对话 Session、MCP OAuth token、工具审批等终端用户流程；它不会降低 API Key 在租户内的 admin 权限。

### 网页嵌入

网页嵌入渠道把某个智能体发布到外部网站。创建渠道时配置：

- 绑定智能体。
- 渠道名称和启用状态。
- 域名白名单。
- 每分钟和每日限流。
- 欢迎语、推荐问题、是否允许联网搜索、是否允许文件上传。
- 页面标题、标题模式、挂件位置、默认语言、主色。
- Webhook URL 和 Webhook Secret。

保存后可复制 iframe、普通 Widget 或安全模式 Widget 代码。普通模式会把发布 Token 暴露到浏览器，生产环境建议使用安全模式：发布 Token 保存在你的业务后端，浏览器只拿短时 `ems_` 会话 Token。

### IM 渠道

IM 渠道把智能体接入企业微信、飞书、Slack、Telegram、钉钉、Mattermost、微信、QQBot 等平台。一个智能体可绑定多个 IM 渠道。

创建时设置：

- 绑定智能体。
- 平台。
- 渠道名称。
- 接入模式：WebSocket、Webhook，微信为长轮询绑定流程，Mattermost 默认 Webhook。
- 输出模式：流式或完整输出。
- 会话模式：按用户或按线程。线程模式主要适合 Slack、Mattermost、飞书、Telegram。
- 文件知识库：用于接收 IM 文件并写入指定知识库。
- 平台凭证：不同平台字段不同。

Webhook 模式保存后会显示回调地址，一般是：

```text
https://你的域名/api/v1/im/callback/{channel_id}
```

需要复制到对应平台的事件回调配置中。

### Chrome 插件

Chrome 插件适合网页侧边栏问答、网页剪藏、Markdown 笔记和快捷使用知识库。集成页面提供 Chrome Web Store 入口；左下角账号菜单还提供离线插件下载、离线安装指南和一键配置。

离线安装要点：

- 使用 Chrome 浏览器。
- 下载 `weknora-chrome-extension.zip` 后必须先解压。
- 在 `chrome://extensions` 开启开发者模式。
- 选择“加载已解压的扩展程序”，不要直接选择 zip。
- 回到 WeKnora 点击“一键配置”，会把当前工作区 API 地址和认证配置写入插件本地配置。

### ClawHub

## 设置中心

设置中心按角色展示不同入口。

### 账号设置

- 通用设置：语言、外观等个人偏好。
- 用户资料：查看当前用户信息。
- API：查看或重置 API Key、复制 API URL。该入口通常需要 owner。

### 工作区设置

- 租户信息：查看和编辑工作区基本信息。
- 成员管理：邀请成员、修改角色、移除成员。
- 对话历史：配置会话历史检索所需的 Embedding 模型等。

### 模型与运行环境

模型类型包括：

- KnowledgeQA：主问答/生成模型。
- Embedding：向量化模型。
- Rerank：重排模型。
- VLLM：视觉语言模型。
- ASR：语音识别模型。

支持的来源包括本地、远程 API、阿里云、智谱、火山、DeepSeek、混元、Minimax、OpenAI、Gemini、Mimo、SiliconFlow、Jina、OpenRouter、NVIDIA、Novita、Azure OpenAI 等。

Ollama 设置用于检测本地 Ollama 地址、查看可用模型和拉取模型。

### 数据与扩展

| 设置 | 用途 |
| --- | --- |
| 向量库 | 管理向量存储连接，测试连接，配置索引参数。 |
| 解析器 | 管理文档解析引擎，例如内置解析或 MinerU 等。 |
| 存储 | 管理文件存储后端，例如本地、MinIO、COS 等。 |
| 联网搜索 | 配置搜索提供商、凭证并测试。 |
| MCP 服务 | 配置 SSE、HTTP Streamable、Stdio MCP 服务，认证、OAuth、工具审批、超时和重试。 |

### 平台管理

SystemAdmin 可见平台级设置：

- 全局设置。
- 默认配置中心：从源用户选择模型、向量库、解析、存储、网络搜索、MCP 服务，下发到用户自己的工作区。
- IAM 同步：配置统一身份认证、组织人员同步、定时同步和手动同步。
- 系统信息：查看系统运行信息。

## MCP 服务

MCP 服务用于把外部工具接入智能体。入口在“设置 → MCP 服务”。

### 创建服务

创建时配置：

- 名称和描述。
- 传输方式：SSE、HTTP Streamable、Stdio。
- URL 或本地命令。
- 自定义 Headers。
- 认证策略：无/API Key 或 Token/OAuth 2.0。
- OAuth scopes 和授权服务器元数据 URL。
- 超时、重试次数和重试间隔。

API Key/Token 类静态密钥通过凭证子资源保存，响应中只显示是否已配置，不返回明文。

### 测试和工具审批

测试连接会返回可用工具和资源。对于危险工具，可以配置“需要审批”。智能体运行到这类工具时，会暂停等待用户审批；拒绝后工具不会执行。

OAuth MCP 服务的 token 按 Principal 隔离。也就是说，同一租户下，不同网页登录用户、API 外部用户、IM 用户或嵌入访客拥有各自的 OAuth 授权。

## 平台管理

### 统一身份认证和组织同步

系统可配置 IAM SSO 登录和组织人员同步。同步计划默认每日运行，也可手动触发。同步会按配置创建、更新或停用本地用户，并记录同步运行结果。

生产环境配置 SSO 时，反向代理必须正确透传 Host、X-Forwarded-Host、X-Forwarded-Proto，否则回调地址可能生成错误，导致登录失败。

### 默认配置中心

默认配置中心用于把一组资源复制或下发到用户自己的工作区，例如模型、向量库、解析器、存储、联网搜索、MCP 服务。支持模板变量：

- `${tenant_id}`
- `${user_id}`
- `${username}`

这适合平台管理员统一给新用户准备默认模型和工具环境。

## 常见问题与排错

### 看不到“使用指南”

先刷新页面或重新登录。如果仍看不到，请确认：

- 后端已经部署并启动包含“使用指南”自动加入逻辑的版本。
- 用户是 active 状态。
- 系统启动或用户 provisioning 没有报错。
- 共享空间或知识库没有被管理员误删或取消共享。

### 创建知识库时提示缺模型

启用了向量或关键词检索时必须选择 Embedding 模型。启用多模态解析时需要 VLM 模型。启用音频解析时需要 ASR 模型。进入设置中心确认模型已创建、启用并可测试通过。

### 上传后一直解析中

检查文档解析时间线、后端任务日志和解析器配置。大文件、扫描 PDF、图片密集文档和音频转写耗时更长。若失败，可调整解析器、关闭不必要的图谱/问题生成、多模态配置后重解析。

### 回答没有引用

可能原因：

- 当前智能体没有启用知识库。
- 知识库选择模式为 none。
- 智能体设置为仅 @ 提及时检索，但问题没有 @。
- 文档未解析完成。
- 检索阈值过高、TopK 太小或问题与资料不匹配。
- 问题被识别为闲聊或非检索意图。

### 回答引用不准

优先检查：

- 知识库分块是否过大或过小。
- 是否启用了合适的 Embedding 和 Rerank 模型。
- FAQ 是否缺少相似问或反例问。
- 同一知识库是否混入无关资料。
- 智能体系统提示词是否要求严格依据引用回答。

### 共享资源看得到但不能编辑

检查四层权限：当前租户角色、资源归属、共享权限、共享空间角色。只读共享、空间 viewer、租户 viewer 都会导致只能查看。

### 打开会话后智能体、模型或上下文不符合预期

当前版本不会因为打开旧会话而自动切回旧智能体或“快速问答”；输入区会保留你当前选择的智能体和模型。旧会话只恢复该会话的资源上下文，包括知识库、文档、标签、MCP、技能和联网搜索。新建会话会清空这些资源上下文。

如果同一会话中的附件或图片没有恢复，先确认是否已经发送过。已发送内容会保留在消息里；尚未发送的本地文件只保存在当前浏览器运行内，刷新页面后需要重新选择。

### API 调用返回 401

检查：

- 是否带了 `X-API-Key` 或 `Authorization: Bearer <token>`。
- API Key 是否属于当前租户，是否被重置。
- signed_token 模式下 `X-External-User-Token` 是否为 HS256、aud 是否为 `weknora`、是否包含 `sub` 和 `tenant_id`、是否过期或超过 24 小时。
- direct_header 模式下是否缺少 `X-External-User-ID`。

### API 调用返回 403

常见原因是当前身份角色不够、目标租户不可访问、共享空间权限不足、资源不属于当前租户且没有共享路径。API Key 在租户内是 admin，但不会获得 SystemAdmin。

### 网页嵌入无法加载

检查渠道是否启用、Token 是否正确、域名白名单是否包含当前 Origin、限流是否触发。如果是安全模式，确认你的取令牌接口返回 `{ token, expiresIn }`，并且服务端 exchange 请求手动带了与白名单匹配的 `Origin`。

### 对话分享链接打不开

先确认已经登录 WeKnora。分享页路径是 `/share/chat/:token`，生产环境需要保证 `/share/` 和 `/assets/` 可以服务桌面 SPA 静态资源，并且 `/api/` 正常代理到后端。分享链接是创建时的快照，不会跟随原会话实时更新。

### IM 渠道没有回复

检查渠道是否启用、平台凭证是否正确、Webhook 回调地址是否公网可达、WebSocket 是否连上、绑定智能体是否可运行、文件知识库是否存在、平台事件权限是否开通。

### MCP 工具不可用

进入设置中心测试 MCP 服务。检查传输方式、URL/命令、认证、OAuth 授权、超时、工具审批状态。OAuth 工具需要当前终端用户完成授权，不同用户不会共用授权。

### 数据分析查询失败

检查数据库源连接、启用表范围、字段描述、敏感级别、最大行数、超时和 SQL 权限。建议在智能体提示词中要求先查看 schema，再查询；不要直接让模型猜表名。

## 推荐维护方式

平台管理员维护“使用指南”知识库时，建议至少放置两份文档：

- 本文档：面向所有用户，覆盖系统怎么用。
- 智能体开发指南：面向智能体配置者和集成开发者，覆盖如何在平台上设计、配置、测试、发布和集成智能体。

更新文档后，建议在“使用指南”知识库中重解析文档，并用常见问题验证检索效果，例如“如何创建 FAQ 知识库”“如何配置 signed token API 集成”“为什么共享知识库不能编辑”。
