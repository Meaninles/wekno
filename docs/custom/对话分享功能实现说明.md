# 对话分享功能实现说明

## 目标与入口

对话分享用于把一个聊天 thread 导出为可复制链接，链接格式统一为：

```text
/share/chat/:token
```

该链接复用现有前端域名，不区分电脑端链接和移动端链接。电脑浏览器打开时展示桌面 Web 分享页，移动设备或窄屏打开时展示移动端分享页。

访问分享页必须具备 WeKnora Web 登录态，但不要求访问者拥有原会话、原租户或原知识库权限。分享页只展示创建分享时的消息快照，不提供继续提问、引用跳转、来源时间线跳转或原会话跳转能力。

## 代码边界

后端二开模块：

- `internal/custom/modules/chatshare/`：分享链接、消息快照、图片/附件代理、产物下载代理和服务逻辑。
- `internal/custom/bootstrap/bootstrap.go`：执行 `chatshare` AutoMigrate 并注册 `/api/v1/custom/chat-share/*` 路由。

前端二开模块：

- `frontend/src/custom/modules/chatshare/`：接口封装、分享图标、分享页、登录回跳、受保护文件 URL 转换和分享消息媒体字段归一化。
- `frontend/src/router/index.ts`：注册 `/share/chat/:token`，并标记为登录态页面。
- `frontend/src/App.vue`、`frontend/src/views/auth/Login.vue`、`frontend/src/utils/request.ts`：保留分享链接登录前后的 `returnTo`。
- `frontend/src/components/menu.vue`、`frontend/src/custom/modules/mobile/views/MobileChat.vue`：挂载电脑端和移动端对话分享按钮。
- `frontend/src/views/chat/components/botmsg.vue`、`frontend/src/views/chat/components/usermsg.vue`、`frontend/src/views/chat/components/tool-results/GeneralAgentArtifactsResult.vue`、`frontend/src/custom/modules/mobile/components/MobileChatMessage.vue`：在分享模式下禁用引用跳转，通过分享文件代理展示图片和附件，并展示可下载的通用智能体产物。

## 后端流程

### 创建分享

```text
POST /api/v1/custom/chat-share/sessions/:session_id
```

创建分享要求当前访问者是正常 Web 登录用户。服务端先复用原生 `SessionService.GetSession` 校验创建者对原会话的访问权限，然后读取当前会话消息并写入快照。

创建时会生成 32 字节随机 token，并只把 `sha256(token)` 写入数据库。明文 token 只在创建响应中返回，用于组装分享链接。

响应中的 `url` 会优先使用后端 `FRONTEND_BASE_URL` 拼成公网绝对地址；未配置时返回 `/share/chat/:token` 相对路径，由前端按当前域名补全。

写入的数据表：

- `custom_chatshare_links`：分享链接元数据、token hash、源租户、源会话、创建人、标题、访问次数和状态。
- `custom_chatshare_messages`：消息快照，包含角色、内容、图片、附件、提及项和安全的结构化图表结果。

快照会清空 `knowledge_references`，避免分享页出现引用跳转。图片、附件和提及项会保留，用于只读展示。分享功能不会向查看者暴露 `agent_steps`，但会从原消息的 `agent_steps` 中抽取 `structured_analysis_result` 的图表渲染数据，单独保存到 `tool_results`。

为兼容旧快照，查看分享时如果快照中的 `tool_results` 为空，服务端会按原消息补齐安全的结构化图表数据，但不会同步原会话新增或删除的消息，也不会返回完整 `agent_steps`。

### 查看分享

```text
GET /api/v1/custom/chat-share/:token
```

查看分享同样要求 Web 登录态。服务端只按 token hash 查找 active 分享链接，不再检查访问者是否拥有原会话权限。读取成功后返回标题和消息快照，并更新 `view_count`、`last_viewed_at`。

返回内容以消息快照为准，除兼容回填结构化图表数据外，不会重新读取原会话最新消息列表。原会话后续新增或删除消息不会自动影响已创建的分享链接。

通用智能体产物按 `custom_general_agent_artifacts.tenant_id + session_id + message_id` 绑定到分享消息。服务端只返回产物文件元信息，并把下载地址改写为分享代理：

```text
GET /api/v1/custom/chat-share/:token/artifacts/:artifact_id/download
```

这样分享页可以展示 ECharts 图表和下载产物文件，但不会显示 agent steps、工具调用过程或原会话跳转入口。

### 分享文件代理

```text
GET /api/v1/custom/chat-share/:token/files?file_path=...
```

分享页中的图片和附件通过该接口读取。服务端会：

- 要求 Web 登录态。
- 校验 token 对应 active 分享链接。
- 拒绝空路径和包含 `..` 的路径。
- 从存储路径中解析租户段，并要求它等于分享链接记录的源租户。
- 优先使用源租户存储配置读取文件；当路径 provider 与全局 `STORAGE_TYPE` 一致且全局文件服务可用时，按当前全局存储配置回退。

这样访问者不需要原会话权限，也不会得到任意文件读取能力。

### 分享产物下载代理

```text
GET /api/v1/custom/chat-share/:token/artifacts/:artifact_id/download
```

分享页中的通用智能体产物通过该接口下载。服务端会：

- 要求 Web 登录态。
- 校验 token 对应 active 分享链接。
- 校验产物属于分享链接记录的源租户和源会话。
- 从本地持久化产物路径读取文件，并以附件方式返回。

该接口只服务分享页的下载动作，不提供产物入库、预览原会话或查看 agent steps 的能力。

## 前端行为

电脑 Web 对话页的分享按钮固定在整个页面右上角，按钮本身无边框、无背景，仅显示图标。移动端分享入口挂在聊天标题区域，优先调用系统分享能力，不支持时复制链接到剪贴板。

分享页复用 `/share/chat/:token` 一个路由，根据视口宽度切换布局：

- 桌面布局使用 Web 消息组件，并复用对话页的右侧横线问题导航 UI。
- 移动布局使用移动端消息组件，适配移动端滚动和安全区。

分享页是只读页面，不渲染输入框、停止生成、继续会话、agent steps、引用列表和来源跳转入口。消息组件通过 `shareMode` 和 `shareToken` 参数进入分享模式，图片和附件 URL 会改写到 `/api/v1/custom/chat-share/:token/files`。结构化分析图表复用对话页的 ECharts 渲染组件展示，通用智能体产物以只读文件列表展示，桌面端隐藏“入库”动作，桌面端和移动端都保留下载按钮。

未登录用户打开分享链接时，会跳转到：

```text
/login?returnTo=/share/chat/:token
```

登录或 SSO 完成后回到原分享链接。

## 部署要求

分享链接必须复用现有域名，生产同域路径分流应满足：

```text
/api/      -> WeKnora 后端 app service
/mobile/   -> 移动端 mobile-web service
/share/    -> 能服务桌面 SPA 的 web service，或已内置桌面 SPA 的 mobile-web service
/assets/   -> 桌面 SPA 静态资源
/          -> 电脑端 web service
```

路径优先级必须保证 `/api/`、`/mobile/`、`/share/`、`/assets/` 优先于 `/`，否则分享页刷新、登录回跳或静态资源加载可能被错误服务接走。

如果希望创建分享接口直接返回公网完整链接，可在 app 侧配置 `FRONTEND_BASE_URL=https://knora.moutai.com.cn`。未配置时接口返回同域相对路径，前端仍会按浏览器当前域名补全并复制。

当前项目的 mobile-web 镜像已经支持直接承载分享页：

- `frontend/Dockerfile.mobile` 同时执行 `npm run build:mobile` 和 `npm run build`。
- `dist` 被复制到 `/usr/share/nginx/html`，用于 `/share/` 和 `/assets/`。
- `dist-mobile` 被复制到 `/usr/share/nginx/html/mobile`，用于 `/mobile/`。
- `frontend/nginx.mobile.conf` 对 `/share/` fallback 到 `/index.html`，对 `/assets/` 提供桌面 SPA 静态资源，对 `/mobile/` fallback 到 `/mobile/mobile.html`。
- `frontend/docker-entrypoint.sh` 生成 `/config.js`，并在存在移动端目录时复制为 `/mobile/config.js`。
- `frontend/.dockerignore` 保留 `index.html`、`embed.html`、`vite.config.ts` 等桌面 SPA 构建入口，避免 mobile-web 镜像中缺少分享页所需资源。

Helm 默认部署下，`mobileWeb.enabled=true` 只会显式增加 `/mobile -> mobile-web`，`/share/` 仍由电脑端 frontend 的 `/` catch-all 承载。这个默认行为可以满足同一分享链接在电脑端和移动端查看的要求，因为分享页自身会按视口切换布局。

如果生产环境把 `/share/` 转给普通电脑端 web service，则 web service 必须支持 SPA fallback 到 `index.html`，并能访问相同的 `/api/` 后端代理。如果把 `/share/` 转给 mobile-web service，则必须同时把 `/assets/` 转给 mobile-web，保证分享页所需的桌面 SPA 静态资源也由同一服务提供。

## 验证清单

修改分享功能或部署路径后，至少验证：

- 电脑端对话页分享按钮固定在页面右上角，按钮视觉为无边框图标。
- 电脑端打开 `/share/chat/:token` 渲染桌面分享页，右侧横线导航与对话页一致。
- 移动端打开同一个 `/share/chat/:token` 渲染移动端分享页。
- 未登录访问分享链接会跳转登录页，并在登录后回到原分享链接。
- 分享页没有输入框、继续会话入口和引用跳转入口。
- 分享页能展示对话页中的结构化分析 ECharts 图表，但不展示 agent steps。
- 通用智能体产物列表可见且可通过 `/api/v1/custom/chat-share/:token/artifacts/:artifact_id/download` 下载。
- 图片和附件通过 `/api/v1/custom/chat-share/:token/files` 正常展示。
- mobile-web 镜像或 Ingress 下直接刷新 `/share/chat/:token` 返回 SPA 页面，而不是 404。
