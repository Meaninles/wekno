# 知识库管理智能体

`knowledge-base-manager` 是一种可在智能体编辑器中选择的自定义智能体类型，不会自动创建任何预定义或内置智能体。它复用通用智能体（Claude Agent SDK）运行时，并在其上增加受控的完整文档新增、先增后删替换和删除工具。

## 配置模型

- 必须使用智能推理模式。
- 必须配置 ReRank 模型。新建时会优先自动选择租户默认 ReRank，没有默认时选择首个可用 ReRank；租户没有可用 ReRank 时会阻止保存并提示先配置模型。
- 知识库范围固定为“指定知识库”，不能选择全部、不能选择不使用知识库，至少选择一个。
- FAQ 知识库不可选；共享知识库要求调用者具有可编辑或更高权限。
- 支持一套默认权限和稀疏的逐知识库覆盖。没有覆盖的知识库继承默认权限。
- 权限只有新增、修改、删除三项。修改是派生权限，始终等价于新增 + 删除；只要缺少其中任一项，运行时就不会注册替换工具。
- 文件类型不做智能体级缩减，交给所选知识库的原生解析器校验，以覆盖原生知识库支持的全部文档格式。

保存配置和每次对话执行时都会重新校验。最终有效权限为：

```text
智能体配置权限 ∩ 本轮 @ 选择范围 ∩ 当前调用者实时平台权限
```

## 本轮范围

- 没有显式 @：使用智能体配置的全部指定知识库。
- @ 知识库：本轮完整文档操作只允许该知识库。
- @ 文档：读取、替换、删除只允许该文档；替换产生的新文档只能进入其原知识库，不能借此新增无关文档。
- 同时 @ 知识库和文档：在智能体配置范围内取这些显式目标的并集，其中知识库保持库级范围、文档保持文档级范围。
- @ 标签：仅作为只读检索条件，不授予任何文档变更能力。
- 显式选择越界、目标不存在或无权限时整轮失败关闭，不回退到智能体更宽的配置范围。

## 文件输入

原生解析只接受两类可信输入：

1. 当前消息上传的附件（`input_file`）；
2. 当前通用智能体运行中由 `create_artifact` 注册的产物（`artifact`）。

不接受 URL、网页地址、文件系统路径、历史轮次 token、知识分片文本或直接选中的知识库原文作为摄取源。若内容来自联网搜索，智能体必须先搜索与核验，生成完整本地产物并注册，再把产物加入知识库。

`knowledge-base-manager` 不设置单次运行的产物数量上限，因而可以在一次管理任务中生成并处理超过 5 个独立文档。该例外只作用于此智能体类型；通用智能体等其它类型仍保持最多 5 个产物。共享的单文件校验和单次返回产物总大小小于 128MB 的限制仍然生效。

## 先增后删替换

修改由智能体编排成两个明确的变更调用：

1. 检查旧文档清单、稳定 ID、文件哈希和元数据；
2. 调用 `kb_replace_document`，从本轮附件或产物把新文档写入旧文档所在知识库；该工具只新增，绝不删除旧文档；
3. 确认工具返回“新文档已添加”；
4. 不等待分片、索引、Wiki 或知识图谱，立即由智能体调用 `kb_delete_document` 删除步骤 1 确认的旧文档；
5. 新文档的原生解析和派生处理继续在后台执行。

后台没有自动删除旧文档的代码路径。新文档写入失败时，智能体不得调用删除；新文档写入成功但独立删除调用失败时，新旧文档会暂时并存，智能体必须如实报告。由于用户选择在“写入成功”而非“解析成功”后删除，后续解析失败不会自动恢复旧文档。

对话不等待后台解析。新增在 `kb_add_document` 确认写入后即可回复；替换则必须再等待独立的 `kb_delete_document` 调用返回，才能报告旧文档是否删除。只有用户明确询问处理状态或要求等待时，智能体才调用 `kb_mutation_status`；“已添加”不等同于“已解析完成”。

## 管理工具

| 工具 | 作用 |
| --- | --- |
| `kb_list_documents` | 在本轮精确范围内列出文档级元数据、哈希、解析状态和标签 |
| `kb_add_document` | 从本轮附件或产物新增完整文档 |
| `kb_replace_document` | 只把替换用的新文档写入旧文档所在知识库，不删除旧文档 |
| `kb_delete_document` | 使用原生清理链路删除完整文档 |
| `kb_mutation_status` | 仅在用户明确询问或要求等待时查询后台处理状态 |

操作记录保存在 `custom_kbmanager_operations`。记录包含调用者、智能体、知识库、文档、文件哈希、状态、原因和时间，不保存文件字节、URL、存储路径或凭据。

通用智能体运行目录还会准备 `generated/kb_manager/`，其中的只读脚本用于检查候选文件哈希和比较文本导出；脚本本身不能修改知识库，所有变更仍必须通过上述管理工具。
