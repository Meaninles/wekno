# 统一身份认证与默认配置二开实现说明

## 目录边界

- 后端二开逻辑位于 `internal/custom/`。
- 前端二开页面位于 `frontend/src/custom/`。
- 自定义迁移参考文件位于 `migrations/custom/`。
- 原生代码只保留必要注册点：
  - `internal/container/container.go` 注册 custom bootstrap。
  - `internal/router/router.go` 挂载 `/api/v1/custom` 路由。
  - `internal/handler/auth.go` 在 `/auth/me` 调用可选 provisioning hook。
  - `internal/middleware/auth.go` 放行 SSO public callback。
  - `frontend/src/views/settings/Settings.vue` 挂载两个设置页入口。
  - `frontend/src/views/auth/Login.vue` 增加可选统一身份认证登录按钮。

## 功能模块

### IAM 登录与组织人员同步

- 后端模块：`internal/custom/modules/iam`
- 管理接口：`/api/v1/custom/iam/*`，仅 SystemAdmin 可访问。
- 登录接口：`/api/v1/custom/iam/sso/*`，浏览器回调前无需 WeKnora token。
- 默认同步计划：每日 `03:10`。
- 同步结果写入二开表 `custom_iam_*`，并按需创建或启停 WeKnora 本地用户。

#### 生产入口与反向代理要求

- IAM 工作台应用入口配置为：`https://knora.moutai.com.cn/api/v1/custom/iam/sso/entry`
- IAM OAuth 回调地址配置为：`https://knora.moutai.com.cn/api/v1/custom/iam/sso/callback`
- 生产环境不需要对外暴露后端 `8080` 端口；`/api/` 请求应从前端域名进入，再由前端 Nginx 或网关代理到后端。
- 如果前端容器前面还有云网关、负载均衡或外层 Nginx，必须向前端层透传下面的请求头：

```nginx
proxy_set_header Host $host;
proxy_set_header X-Forwarded-Host $host;
proxy_set_header X-Forwarded-Proto $scheme;
```

原因：SSO `entry` 会根据外部访问域名和协议动态生成发给 IAM 的 `redirect_uri`。`X-Forwarded-Host` 用于还原公网域名，`X-Forwarded-Proto` 用于还原公网协议；如果 HTTPS 在外层终止但没有传 `X-Forwarded-Proto: https`，后端可能生成 `http://.../callback`，导致 IAM 回调地址不匹配或浏览器被错误跳回 HTTP。

### 默认配置中心

- 后端模块：`internal/custom/modules/configcenter`
- 管理接口：`/api/v1/custom/config-center/*`，仅 SystemAdmin 可访问。
- 管理员从源工作区选择模型、向量数据库、解析、存储、网络搜索、MCP 服务。
- 配置下发方式是复制到用户自己的工作区，尽量复用原生 WeKnora 的资源选择和运行逻辑。
- 支持模板变量：`${tenant_id}`、`${user_id}`、`${username}`。

## 运行注意

- 开发环境启动时通过 custom bootstrap 执行 GORM AutoMigrate，避免改动原生 migration runner。
- 生产发布如需显式 SQL 迁移，可参考 `migrations/custom/` 下的 SQL 文件。
- 若多个用户共享同一个 WeKnora tenant，原生资源可见性仍是 tenant 级；本实现默认按用户 home tenant 下发配置。
