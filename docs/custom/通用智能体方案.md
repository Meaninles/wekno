## 运行时覆盖范围

当前 Claude SDK 运行时覆盖四个 `agent_type`：

| `agent_type` | Go 运行器 | Python 旁路服务 | 前端主要能力 | 数据库源要求 |
| --- | --- | --- | --- | --- |
| `general-agent` | `generalagent.Service.Run` | `custom/services/general-agent` | 知识库、联网、MCP、轻量技能、专业技能、数据库、产物生成 | 可选 |
| `document-processing-agent` | `generalagent.Service.Run` | `custom/services/document-processing-agent` | 文档模板、附件、专业技能、数据库、产物生成 | 可选 |
| `data-analysis` | `generalagent.Service.Run` | `custom/services/general-agent` | 数据库分析、SQL、图表、RAG 辅助解释 | 必填 |
| `table-analysis` | `generalagent.Service.Run` | `custom/services/general-agent` | CSV/Excel 表格分析、DuckDB SQL、结构化图表、RAG 辅助解释 | 不使用 |

`types.IsClaudeSDKAgentType` 对这四个类型返回 `true`。后端保存智能体时，只有 `data-analysis`、`general-agent` 和 `document-processing-agent` 会保留 `db_data_sources`；其中只有 `data-analysis` 在校验和前端 ready 状态中要求至少绑定一个数据库数据源。`table-analysis` 不保存数据库源，预设通过 `supported_file_types` 限制 CSV/Excel 文件类型。

## 主链路

1. 对话请求进入原生会话链路后，后端根据智能体配置选择运行器。
2. `internal/custom/bootstrap.NewHandlers` 注册 `general-agent`、`document-processing-agent`、`data-analysis`、`table-analysis` 四个类型的运行器。
3. `generalagent.Service.Run` 复用 `SessionService.BuildAgentRuntimeConfig` 构建运行时配置，继续沿用原生模型、知识库、检索、联网、MCP、技能、附件和工具注册逻辑。
4. Go 侧用 `AgentService.CreateToolRegistry` 创建本次运行的工具注册表，再把工具结构、运行时配置、上下文、历史消息、附件、专业技能包和 LLM 配置下发给 Python 旁路服务。
5. Python 旁路服务用 Claude Agent SDK 执行智能体推理循环。它只负责模型循环、提示词组织、动态工具声明和产物本地生成。
6. Python 调用工具时统一回调 Go：`POST /api/v1/custom/general-agent/internal/tools/call`。
7. Go 只允许执行本次 run 已注册的工具，并在 Go 侧完成租户、权限、MCP OAuth/审批、知识库检索、数据库查询、Web Search、审计和来源引用整理。
8. Python 通过 NDJSON 流返回 `answer_delta`、`thinking`、`progress`、工具事件和产物信息，Go 转成 WeKnora 原生流事件并持久化消息、步骤和产物。

Python 旁路服务不直接连接 WeKnora 数据库、对象存储、MCP 服务或业务数据库。Go 后端仍是权限、密钥、租户隔离、工具执行和持久化边界。

## 配置字段

运行时配置以 `CustomAgentConfig` 为源，默认值由 `EnsureDefaults` 和类型预设补齐。

### 基础与模型

| 字段 | 当前实现 |
| --- | --- |
| `agent_mode` | Claude SDK 类型仍使用 `smart-reasoning` 模式。 |
| `agent_type` | 决定是否进入 Claude SDK 运行器以及选择哪个旁路服务。 |
| `system_prompt` / `system_prompt_id` | 后端渲染后传给旁路服务，作为系统提示词核心内容。 |
| `context_template` / `context_template_id` | 主要用于 RAG/quick-answer 上下文模板；Claude SDK 运行时会携带可见上下文摘要。 |
| `model_id` | 从 WeKnora 模型管理读取对话模型、凭据、provider、base URL 和 `extra_config`。 |
| `temperature` | 写入运行时配置和提示约束。 |
| `max_completion_tokens` | 写入运行时配置。 |
| `thinking` | 写入运行时配置；旁路服务根据 SDK 能力处理。 |
| `max_iterations` | 传给旁路服务作为最大轮数。默认值为 10，部分类型预设会覆盖。 |
| `llm_call_timeout` | 控制 Go 到旁路服务、旁路服务到模型的长任务超时。 |
| `enable_artifacts` | 控制是否暴露产物工具。 |

模型适配由 `generalagent.resolveLLMConfig` 完成。Anthropic/Claude provider 可直接使用模型配置；普通或本地 OpenAI-compatible 模型如果要给 Claude SDK 使用 Anthropic 兼容接口，需要在模型 `extra_config` 中设置：

- `general_agent_claude_base_url`、`general_agent_claude_endpoint` 或 `claude_base_url`
- `general_agent_claude_model_name`、`general_agent_claude_model` 或 `claude_model_name`
- `general_agent_claude_provider` 或 `claude_provider`

代码没有用全局环境变量覆盖每次运行的 Claude 模型名和 base URL。不同智能体应通过各自选择的 WeKnora 模型配置隔离。

### 知识库与检索

| 字段 | 当前实现 |
| --- | --- |
| `kb_selection_mode` | `all` / `selected` / `none` 控制可见知识库范围。 |
| `knowledge_bases` | `selected` 模式下绑定知识库。共享智能体使用智能体源租户解析范围。 |
| `retrieve_kb_only_when_mentioned` | 开启后，仅在用户显式 @ 知识库、文档或资源时检索。 |
| `retain_retrieval_history` | 控制检索历史保留策略。 |
| `embedding_top_k`、`keyword_threshold`、`vector_threshold`、`rerank_top_k`、`rerank_threshold` | 传入原生检索工具和 RAG 工具。 |
| `rerank_model_id` | 在 `knowledge_search` 可用且配置了 rerank 模型时加载。 |
| FAQ 相关字段 | 继续由原生检索/FAQ 逻辑使用。 |

知识库整体选择仍走检索和工具链路。只有对话中显式上传的文件、图片、音频，以及用户明确选中的知识库具体文件，会额外进入 Claude SDK 原文件副本链路。

### 联网搜索

| 字段 | 当前实现 |
| --- | --- |
| `web_search_enabled` | 与本次请求的 `WebSearchEnabled` 同时为 true 时启用 Web Search。 |
| `claude_sdk_web_search_enabled` | Claude SDK 类型的原生联网开关，同样要求本次请求启用联网。 |
| `web_search_provider_id` | 为空时使用租户默认搜索提供商。 |
| `web_search_max_results` | 控制搜索结果数量。 |
| `web_fetch_enabled` / `web_fetch_top_n` | 控制网页正文抓取工具及抓取数量。 |

联网搜索和网页抓取的 provider、凭据和 SSRF 校验都在 Go 侧执行，旁路服务不保存搜索密钥。

### MCP

| 字段 | 当前实现 |
| --- | --- |
| `mcp_selection_mode` | `all` 使用全部可用服务；`selected` 只使用 `mcp_services`；`none` 禁用；未知值按禁用处理。 |
| `mcp_services` | `selected` 模式下的服务 ID。空列表不会退化为 `all`。 |
| `mcp_auth_wait_timeout` | OAuth 授权等待超时。 |

MCP 工具由 Go 侧注册并回调执行。OAuth、审批、transport、密钥和工具输出处理继续复用原生 MCP 链路。

### 技能

当前项目有三类技能概念：

| 类型 | 配置/来源 | 当前行为 |
| --- | --- | --- |
| 轻量技能 | `lightweight_skills_selection_mode`、`selected_lightweight_skills`；旧字段 `skills_selection_mode`、`selected_skills` 仍作为兼容回退 | 作为提示词/上下文能力注入，可在技能中心管理、共享和对话中临时选择。 |
| 预加载运行时技能 | `skills/preloaded/` | 原生 Agent skills 目录，启用时通过 `read_skill` / `execute_skill_script` 走渐进式读取和沙箱执行。 |
| 专业技能 | `professional_skills_selection_mode`、`selected_professional_skills`；`internal/custom/modules/skillhub` 管理 | Claude SDK 运行时会把选中的专业技能包内容传给旁路服务，旁路服务放在 `.claude/skills/<name>` 下供模型按需使用。 |

前端当前只在 `general-agent` 和 `document-processing-agent` 的技能页展示专业技能选择。`data-analysis` 和 `table-analysis` 不展示工具、MCP、技能配置页。

### 数据库数据源

数据库源来自 `internal/custom/modules/dbanalytics`，支持 MySQL 和 PostgreSQL。

| 字段 | 当前实现 |
| --- | --- |
| `db_data_sources` | 限定 `db_catalog`、`db_schema`、`db_query` 可访问的数据源。 |
| `allowed_tools` | 同时控制数据库工具是否暴露。未允许的 DB 工具不会注册。 |

`custombootstrap.NewHandlers` 通过 runtime tool hook 注册数据库工具。注册条件是：

- 智能体类型是 `data-analysis`、`general-agent` 或 `document-processing-agent`。
- 配置了 `db_data_sources`。
- `allowed_tools` 允许对应工具名。

`db_query` 保持只读限制和数据源安全范围。图表能力在这些 Claude SDK 数据库工具使用方中开启。共享智能体运行时使用智能体源租户的数据源范围。

### 表格文件分析

`table-analysis` 不使用 `db_data_sources`。它通过 `table_schema` 和 `table_analysis` 分析知识库或本轮上传的 CSV/Excel 文件，运行时使用 DuckDB 执行 SQL，并在需要展示图表时走结构化图表和 `final_answer` 校验链路。前端该类型隐藏数据库、工具、MCP、技能页，预设会限制 `supported_file_types` 为 `csv`、`xlsx`、`xls`。

### 多模态、附件与原文件副本

| 字段 | 当前实现 |
| --- | --- |
| `image_upload_enabled`、`vlm_model_id`、`image_storage_provider` | 控制图片上传、图片描述和图片存储。 |
| `audio_upload_enabled`、`asr_model_id` | 控制音频上传和 ASR。 |
| `supported_file_types` | 控制可上传/可处理文件类型。 |

Claude SDK 类型会额外准备原文件副本：

- 对话中上传的文件、图片、音频。
- 用户明确选中的知识库具体文件。

Go 后端负责把原文件保存到对象存储，生成临时下载信息；Python 在 run 目录的 `input_files/` 下下载并校验 `size` 和 `sha256`。知识库整体选择不进入原文件副本链路，仍通过检索/工具访问。

相关环境变量：

- `CUSTOM_GENERAL_AGENT_ORIGINAL_INPUT_MAX_MB`，默认 200MB。
- `CUSTOM_GENERAL_AGENT_ORIGINAL_INPUT_STORAGE_PROVIDER`，显式指定 `minio` 或 `obs`。
- `CUSTOM_GENERAL_AGENT_ORIGINAL_INPUT_BUCKET`
- `CUSTOM_GENERAL_AGENT_ORIGINAL_INPUT_PATH_PREFIX`，默认 `weknora/__weknora_claude_sdk_original_inputs__/`。

### 产物

`enable_artifacts` 开启后，旁路服务暴露内部产物工具。实现约束：

- 最多返回 5 个产物。
- 返回给前端持久化的产物合计小于 128MB。
- 产物先写入旁路服务运行目录，再由 Go 下载、去重、落库并提供下载 URL。
- Go 侧下载地址为 `/api/v1/custom/general-agent/artifacts/:id/download`。
- 嵌入页下载地址为 `/embed/.../sessions/:session_id/artifacts/:id/download`。
- 产物记录写入 `custom_general_agent_artifacts`，并在 `agent_steps` 中记录 `create_artifact` 伪工具结果，前端以 `general_agent_artifacts` 卡片展示。

`document-processing-agent` 使用文档处理镜像，额外预装 LibreOffice、Pandoc、PDF/Office 工具库和中文字体。其提示词和运行器会要求在创建产物前进行检查和修正，用于办公文档质量控制。

## 工具注册规则

工具注册由原生 `AgentService.registerTools` 和二开 runtime hook 共同完成。

- `allowed_tools` 为空时使用默认工具集合。
- 没有知识库或搜索目标时，会过滤 RAG/Wiki/表格分析相关工具；`table-analysis` 有本轮上传 CSV/Excel 附件时会保留 `table_schema`、`table_analysis`。
- 没有知识库、没有 Web、没有数据库源时，会禁用 `todo_write`。
- `web_search`、`web_fetch` 只有在联网配置和请求开关都允许时注册。
- MCP 工具按 MCP 选择模式注册。
- 数据库工具由 dbanalytics runtime hook 注册。
- Python 回调只能执行本次运行已注册的工具，不能传任意工具名绕过授权。

工具来源会被归类为 `mcp`、`database`、`wiki`、`skill`、`knowledge`、`web` 或 `native`，用于侧边栏展示、日志和来源引用。

## 旁路服务与环境变量

开发环境 Compose 中有两个旁路服务：

| 服务 | 默认容器名 | 宿主端口 | Go 侧默认 URL |
| --- | --- | --- | --- |
| 通用智能体 | `weknora-custom-general-agent` | `127.0.0.1:8091` | `http://weknora-custom-general-agent:8091` |
| 文档处理智能体 | `weknora-custom-document-processing-agent` | `127.0.0.1:8093` | `http://weknora-custom-document-processing-agent:8091` |

关键环境变量：

- `CUSTOM_GENERAL_AGENT_URL`
- `CUSTOM_DOCUMENT_PROCESSING_AGENT_URL`
- `CUSTOM_GENERAL_AGENT_API_KEY`
- `CUSTOM_GENERAL_AGENT_TOOL_CALLBACK_URL`
- `CUSTOM_GENERAL_AGENT_HTTP_TIMEOUT_SEC`
- `CUSTOM_GENERAL_AGENT_TOOL_EXEC_TIMEOUT_SEC`
- `CUSTOM_GENERAL_AGENT_ARTIFACT_ROOT`
- `GENERAL_AGENT_RUN_ROOT`
- `CUSTOM_GENERAL_AGENT_CLAUDE_API_TIMEOUT_MS`
- `CUSTOM_GENERAL_AGENT_CLAUDE_IDLE_TIMEOUT_MS`
- `CUSTOM_GENERAL_AGENT_MAX_TURNS`

生产 Docker Compose 部署时，`CUSTOM_GENERAL_AGENT_TOOL_CALLBACK_URL` 默认应指向 `http://app:8080/api/v1/custom/general-agent/internal/tools/call`，`CUSTOM_GENERAL_AGENT_ARTIFACT_ROOT` 建议放在 app 持久化卷内，例如 `/data/files/general-agent-artifacts`。如果旁路服务使用 `custom/docker-compose.general-agent.yml` 接入生产主栈，需要设置 `CUSTOM_GENERAL_AGENT_NETWORK=weknora_WeKnora-network`。

内部工具回调接口虽然在全局认证白名单中放行，但 handler 仍校验 `CUSTOM_GENERAL_AGENT_API_KEY`，支持 Bearer token 或 `X-API-Key`。

## 前端呈现

`frontend/src/views/agent/AgentEditorModal.vue` 当前规则：

- `data-analysis`、`general-agent`、`document-processing-agent` 显示数据库数据源页。
- `data-analysis` 和 `table-analysis` 隐藏工具、MCP、技能页。
- `general-agent` 和 `document-processing-agent` 显示工具、MCP、技能页。
- 专业技能页仅对 `general-agent` 和 `document-processing-agent` 显示。
- 文档模板页仅对 `document-processing-agent` 显示。
- 产物开关仅对 `general-agent` 和 `document-processing-agent` 显示。
- 当类型切换到不支持数据库源的类型时，前端会清空 `db_data_sources` 并移除数据库工具。

## API 路由

二开路由统一挂载在 `/api/v1/custom` 下：

- `/api/v1/custom/iam/sso/*`：SSO public 入口。
- `/api/v1/custom/general-agent/internal/tools/call`：旁路服务内部工具回调。
- `/api/v1/custom/config-center/*`：默认配置中心，仅系统管理员。
- `/api/v1/custom/iam/settings`、`/sync`、`/sync-runs`：IAM 设置与同步，仅系统管理员。
- `/api/v1/custom/builtin-agent-defaults/agents/:id/reset`：恢复内置智能体默认配置。
- `/api/v1/custom/skills/*`：轻量技能、专业技能、共享、下载。
- `/api/v1/custom/scheduled-chat/*`：定时会话。
- `/api/v1/custom/db-analytics/*`：数据库源、元数据、共享、智能体绑定。
- `/api/v1/custom/general-agent/artifacts/:id/download`：产物下载。
- `/api/v1/custom/answer-feedback/*`：回答反馈。

嵌入页产物下载由 `RegisterEmbedRoutes` 挂到 `/embed/.../sessions/:session_id/artifacts/:id/download`。

## 当前实现不包含

- Python 旁路服务直连 WeKnora 数据库、对象存储、MCP 服务或业务数据库。
- 给飞书、Notion、语雀、RSS 等内容型同步源新增独立 agent 选择字段；这些内容仍应先同步进知识库，再由知识库选择和检索策略控制。
- 通用无限制代码解释器。当前产物工具不等同于可执行任意 Python 的沙箱。
- `data-analysis` 和 `table-analysis` 的工具、MCP、技能手工配置页。

## 验证建议

变更运行时代码后建议至少验证：

1. `general-agent` 普通对话、知识库检索、联网、MCP selected/all/none、轻量技能、专业技能、数据库源和产物下载。
2. `document-processing-agent` 文档模板、附件、专业技能、产物生成和下载。
3. `data-analysis` 未绑定数据库源时不可用，绑定后可执行 catalog/schema/query 和图表分析。
4. `table-analysis` 可基于知识库或上传的 CSV/Excel 文件执行 table_schema/table_analysis 和图表展示。
5. 共享智能体运行时不泄露调用者私有知识库、MCP、技能或数据库源。
6. 原生 `quick-answer`、`smart-reasoning`、RAG/Wiki/混合类型不回归。
